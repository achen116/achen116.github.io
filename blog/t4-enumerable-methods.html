Enumerable#cycle

Learn about the Ruby Enumerable method: cycle

First of all, what does Enumerable#cycle even mean? This syntax is just a Rubyist's (aka Ruby programmer) way of referring to a method: cycle that is defined in the module: Enumerable. The Enumerable module provides a set of methods that allow us to loop, change, search, or sort through collections, such as arrays, in Ruby; this description might give you a hint as to what the cycle method does. The # symbol identifies the method as an instance method. If you're not familiar with instance methods, don't worry about it now. We'll just be focused on what the cycle method can do for your code in this post :)


Let's begin with an example of Enumerable#cycle to better demonstrate this method.

counting_sheep = ["1 sheep", "2 sheep", "3 sheep", "zzz"]
counting_sheep.cycle(3) { |x| puts x }

The example above would output:

1 sheep
2 sheep
3 sheep
zzz
1 sheep
2 sheep
3 sheep
zzz
1 sheep
2 sheep
3 sheep
zzz


In this example, our array counting_sheep was repeated back to us three times. How did we do this?

  1. First, we called the method cycle on the array counting_sheep
  2. Next, we passed in a parameter value of 3, which tells cycle how many times to repeat. (If we did not pass any parameter value to cycle, it would repeat forever and ever..)
  3. Finally, we defined a code block (in curly brackets {}) which tells cycle what to do with our array. The code parameter |x| refers back to the array, so puts x is actually saying puts counting_sheep.


Based on the example, we can see that Enumerable#cycle repeatedly calls a code block on an object a specified number of times. As the name suggests, this method is most handy when you need to "cycle" through some object (most likely an array) and apply a behavior to it multiple times. It helps programmers adhere to the DRY (Don't Repeat Yourself) rule by preventing the need to print or puts the same line of code "X" amount of times.
