Arrays vs. Hashes

As you may already know, the Ruby language uses variables (a word or name) to hold a single value, like so:

dinosaur = "Tyrannosaurus Rex"

However, what if we need to store a list of different types of dinosaurs? We could create one variable for each type of dinosaur but that can take up a lot of space in your code and get tiresome if we need to access all of the dinosaurs together. Thankfully, Ruby has other options for holding multiple values.

Arrays

Arrays hold multiple objects in a single variable; they act like a list. The list of items is enclosed in square brackets ([ ]) and each item in the list is separated by a comma (,). Like variables, the name of the array should begin with a lowercase letter and is set equal to your list items, like so:

dinosaur_list = ["Tyrannosaurus Rex", "Brachiosaurus", Triceratops", "Velociraptor"]

Array values can include integers, strings, booleans (True/False), and even arrays! Additionally, arrays don't have to be all integers or all strings; they can include a mixture of these data types, like so:

dinosaur_mix = ["Tyrannosaurus Rex", 2, ["3", 4, "Velociraptor"]]

So, what if you want to access just one item in your array? You can access individual values of an array using a numbered index. Programming languages like to start counting at 0. So, the first item in your array will begin with an index 0, like so:

dinosaur_list[0] #returns "Tyrannosaurus Rex"
dinosaur_list[1] #returns "Brachiosaurus"
dinosaur_list[2] #returns "Triceratops"
dinosaur_list[3] #returns "Velociraptor"

Beginning your count at 0 may seem weird at first, but you'll get used to it with some practice! Since array values are accessed using a numbered index, the order of these values will matter when you're trying to call them. However, what if the order of your list doesn't matter?


Hashes

Hashes are similar to arrays in that they hold multiple values in a single variable. However, these values are a collection of key-value pairs, so the order of them doesn't matter. Below is an example of a hash:

dinosaur_hash = {
"Tyrannosaurus Rex" => "T-rex",
"Brachiosaurus" => "Long Neck",
"Triceratops" => "Three Horned"
}

In the example above, 'Tyrannosaurus Rex' is the key and 'T-rex' is the value of the key. I imagine it like a dictionary - in the dictionary, there is a vocabulary word (the key) paired with a definition (the value); these two things are a unit. In this case, the scientific name of the dinosaur is paired with the common nickname, which is easier to say and type! While you access values of an array using a numbered index, you can access values of a hash through their keys, meaning order doesn't matter.

dinosaur_hash["Brachiosaurus"] #returns "Long Neck"

As you can see, arrays and hashes are important, useful tools in Ruby because they hold any number of values in a single variable. There are numerous methods you can apply to arrays and hashes, which will iterate through each of its values and help us follow the DRY (Don't Repeat Yourself) principle.
