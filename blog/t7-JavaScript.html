Variable Differences

How to declare variables in JavaScript vs Ruby

In both JavaScript and Ruby, a variable is used to save some value (i.e. a number, string, or boolean) in a programmer's code. When a value is assigned to a variable, that value can be called using the variables name. So, instead of typing a super long word out like "supercalifragilisticexpialidocious" multiple times, you can assign that string to a variable name like mary_poppins. Then, whenever you write mary_poppins, the program will know to actually substitute in the string value "supercalifragilisticexpialidocious", saving your fingers and brain a lot of work!


Now, what is the difference you ask? The difference is in the syntax; compare the two below.


In JavaScript, variable assignment begins with the var keyword, followed by the variable name set equal to the value. The var keyword creates the variable in its current scope. The semi-colon (;) at the end of the line also tells JavaScript that is the end of the statement. In Ruby, the var keyword and the semi-colon are not necessary; it only requires the variable name set equal to the value. Additionally, JavaScript convention prefers variables with two or more words to use camel case, while Ruby convention prefers inserting an underscore between additional words. However, it is still acceptable to use underscores instead of camel case in JavaScript if you prefer. Just be sure the variable name begins with a lower case letter and not a number in both JavaScript and Ruby.


So, do I have to type var when using that variable in other parts of my code? Nope! (That could get annoying). Reassigning variable values and referencing variables in later code is the same in JavaScript and in Ruby.


In both examples, we receive the same output by calling both variables in a similar way. console.log() and puts are just JavaScript's and Ruby's way of printing out things to the console. As you might guess, there are many more differences between JavaScript and Ruby but don't stress out! Just take in each concept one at a time like this one, and it'll be easy-peasy-lemon-squeezy ;)
